# Submission Checklist

Use this checklist before submitting or presenting the project.

## Code

- [ ] Frontend starts with `npm run dev`
- [ ] Backend targets are present in CMake
- [ ] Route demo executable runs
- [ ] Admin CLI executable runs
- [ ] Route tests pass
- [ ] Admin CRUD tests pass
- [ ] Persistence tests pass

## Data

- [ ] `database/stations.json` is present
- [ ] `database/connections.json` is present
- [ ] `database/history.json` is present
- [ ] Exported JSON files are valid

## Documentation

- [ ] README includes setup steps
- [ ] README includes demo flow and viva talking points
- [ ] Project summary is present
- [ ] Algorithm explanations are present
- [ ] Diagrams are present

## Presentation

- [ ] Screenshots are added to the README
- [ ] Sample output is verified
- [ ] Route path explanation is ready
- [ ] DSA complexity answers are prepared
